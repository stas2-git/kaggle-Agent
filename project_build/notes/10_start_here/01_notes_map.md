# Notes Map

This folder is working context, not the canonical project specification.

## Read Order

1. Use `02_repository_pyramid.md` when you need the repository-level map.
2. Read the summaries in `20_summaries/`.
3. Open `30_full_notes/` only when you need the detailed work-session history.
4. Treat `spec_files/` as the source of truth when notes and specs disagree.

## What The Notes Are For

- Preserve the reasoning behind ADK alignment and architecture choices.
- Keep long work-session plans available without forcing every reviewer or LLM to read them first.
- Explain why the project chose local reproducibility, deterministic tools, and careful security boundaries.

## What The Notes Are Not

- They are not the public Kaggle submission.
- They are not a replacement for the canonical specs.
- They are not raw codelab source material.

## Fast Routing

- Need the current project contract: use `spec_files/`.
- Need what changed during build sessions: use `20_summaries/`.
- Need exact phase-by-phase ADK work history: use `30_full_notes/01_capstone_adk_expansion_plan.md`.
