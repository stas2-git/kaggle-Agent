# Frontend Spec

## Overview

Build the codelab's standalone FastAPI manager dashboard for paused expense-agent sessions. The dashboard lists unresolved `adk_request_input` calls, lets managers approve or reject them, and resumes the matching Agent Runtime session.

## Modes

- `mock`: local demo data and in-memory actions, requiring no cloud resources.
- `vertex`: real `VertexAiSessionService` and Agent Runtime SDK calls using environment configuration and ADC.

## Endpoints

- `GET /` serves the dashboard.
- `GET /api/pending` lists unresolved approvals.
- `POST /api/action/{session_id}` resumes a session.
- `GET /healthz` reports service health and mode.

## Constraints and Safety

- Always use `user_id="default-user"` for the codelab sessions.
- Match function responses to their exact interrupt IDs.
- An approval action must be idempotent from the UI perspective.
- Do not change or duplicate the expense agent's business logic.
- No cloud deployment, API enablement, IAM change, or Pub/Sub provisioning without explicit approval.
- Public unauthenticated manager actions are demonstration-only; production should use IAP or another authenticated manager boundary.

## Success Criteria

- Local dashboard renders pending cards and empty state.
- Approve/reject shows a loading state and final-result modal.
- Pure session scanning correctly distinguishes unresolved and resolved interrupts.
- API tests pass.
- Cloud adapter imports lazily and fails clearly when configuration is absent.
- Container build inputs and Terraform infrastructure definitions are present.
