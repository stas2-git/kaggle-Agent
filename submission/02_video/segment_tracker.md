# Segment Tracker

Not narration content — none of the pipeline scripts read this file, only
`narrative/slide_narration_segments.yaml`. This is a staleness dashboard: does what the
video currently says still match the codebase, and does the recorded audio still match
the script. Read this before deciding whether `draft_demo_video.mp4` needs regenerating.

**Last full content audit: 2026-07-05.**

**Two things can each be stale independently:** the *script* text in
`narrative/slide_narration_segments.yaml`, and the *recorded audio* in `audio/*.mp3`.
Fixing the script does not retroactively fix the audio — Gemini has to actually re-read
the new words, which costs quota (~10/day). The table below tracks both separately.

## How to re-verify

```bash
cd project_build
uv run pytest -q                                                          # segment 6: test count
uv run python3 -m portfolio_agent.run \
  --input tests/golden/loss_ratio_spike.csv --latest-month 2026-06 \
  --force-offline                                                          # segments 4/5: anomaly details
```

If any number changes: update `narrative/slide_narration_segments.yaml` first, mark the
segment above, then test wording cheaply with
`backend/audio_generation/generate_say_preview.py` + `backend/assemble_video.py --segments N`
(free, unlimited) before spending Gemini quota on
`backend/audio_generation/generate_gemini_tts.py`. If the drift also touches the "Live Demo"
evidence cards (segments 4/5/6), re-run `backend/build_slides.py` too - that's the only stage
that regenerates those.
