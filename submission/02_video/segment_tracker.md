# Segment Tracker

This is a staleness dashboard for the demo video. The canonical story contract is
`story/slide_story.yaml`; active slide, caption, local-preview, and Gemini TTS scripts read
that file directly. `story/narration_segments.yaml` is only a compatibility snapshot for
humans and older tooling.

**Last full content audit: 2026-07-06.**

## Stability Rules

- **Stable story/audio script** belongs in `story/slide_story.yaml` under `audio_script`.
- **Current audio files** belong in `story/audio/current/`.
- **Prior audio files** belong in `story/audio/versions/`; the preserved pre-story-contract
  Gemini take is in `story/audio/versions/20260706-pre-story-contract-gemini/`.
- **Volatile facts** such as test counts, eval counts, run IDs, exact demo metric movements,
  report paths, and trace paths should stay visual-only, generated by `backend/build_slides.py`
  from evidence outputs.
- If volatile facts change, usually regenerate slides and assemble the video; do not re-record
  Gemini audio unless `audio_script` itself changed.

## Re-Verification

When code or evals change, refresh the visual evidence:

```bash
cd project_build
uv run pytest -q
uv run python3 -m portfolio_agent.run \
  --input tests/golden/loss_ratio_spike.csv --latest-month 2026-06 \
  --force-offline
```

If only visual facts drift, re-run `backend/build_slides.py` and `backend/assemble_video.py`.
If an `audio_script` in `story/slide_story.yaml` changes, preview it cheaply with
`generate_say_preview.py` before spending Gemini quota.
