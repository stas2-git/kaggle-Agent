# Agent Spec

## Overview

Build a separate ADK 2.0 ambient-expense workflow and prepare it for source-based deployment to Google Agent Runtime. Expenses under $100 are auto-approved in deterministic Python. Expenses of $100 or more receive Gemini risk analysis and pause for a human decision with `RequestInput`.

## Example Use Cases

- A $50 meal is approved without an LLM call.
- A $150 client dinner is reviewed by Gemini and paused for manager approval.
- The same graph runs locally and through the generated Agent Runtime wrapper.

## Tools Required

- Google ADK 2.x graph workflow API.
- Gemini `gemini-3.1-flash-lite` for high-value risk review.
- Agents CLI deployment scaffolding for `agent_runtime`.
- `uv` lock/export for deterministic source packaging.

## Constraints & Safety Rules

- The $100 policy threshold remains deterministic code.
- High-value expenses can never auto-approve.
- Agent Runtime deployment files must not alter core workflow behavior.
- Secrets and project IDs are supplied through environment/deploy configuration.
- No cloud APIs, resources, or deployment may be changed without explicit approval.
- Agent Runtime is request-driven; actual Pub/Sub/Eventarc ambient triggers require Cloud Run or GKE.

## Success Criteria

- Local low-value and high-value graph paths work.
- Deterministic tests pass.
- Behavioral traces cover automatic and HITL routes.
- Agents CLI recognizes `agent_runtime` as the deployment target.
- Dependency lock and Agent Runtime wrapper are present.
- `agents-cli deploy --dry-run` succeeds when a valid Google Cloud project/ADC is supplied.
- The project is ready for, but not actually sent to, Agent Runtime.
