# Agent Spec

## Overview

Build the codelab's local ADK 2.0 ambient expense agent. It receives expense JSON directly or in a Pub/Sub envelope, deterministically auto-approves expenses under $100, and sends expenses of $100 or more through a security screen, Gemini risk review when safe, and human approval.

## Example Use Cases

- A $45 team lunch is auto-approved without an LLM call.
- A $150 software license is reviewed by Gemini and paused for a human decision.
- A high-value expense containing PII is redacted before any model, log, or approval payload receives it.
- A high-value prompt-injection attempt bypasses Gemini and goes directly to human review as a security event.

## Tools Required

- Google ADK 2.0 graph workflow API.
- Gemini model `gemini-3.1-flash-lite` for high-value risk analysis only.
- FastAPI/ADK web server with Pub/Sub trigger support.
- Local Agents CLI evaluation and trace grading.

## Constraints & Safety Rules

- The $100 threshold and routing must remain deterministic Python code.
- Expenses under $100 must never invoke Gemini.
- SSNs and credit-card numbers must be redacted before model input, logs, and human-review payloads.
- Prompt-injection attempts must bypass Gemini, be flagged, and never auto-approve.
- The project is a local prototype; do not deploy or create cloud infrastructure.
- Secrets belong in `.env`, which must not be committed.

## Success Criteria

- Low-value, high-value, PII, and injection scenarios follow the expected routes.
- The ambient endpoint accepts Pub/Sub push envelopes and normalizes subscription names.
- Deterministic tests pass without cloud credentials.
- Five evaluation scenarios generate inspectable traces.
- LLM-as-judge grading is configured and can run once credentials are present.
