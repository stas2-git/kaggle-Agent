# Agent Spec

## Overview

Build the codelab's local ADK 2.0 retail shopping assistant and secure-development lifecycle. The agent redeems single-use discount codes for registered users. The repository demonstrates security-first planning, deterministic tool guardrails, STRIDE threat modeling, outcome-based tests, command-execution validation, and Semgrep/pre-commit secret scanning.

## Example Use Cases

- A registered user redeems `WELCOME50` once successfully.
- A second redemption of the same code is rejected.
- Unknown codes and guest users are rejected without model discretion.
- A destructive development command is blocked by the local agent hook.
- A credential-shaped string is detected by the Semgrep gate and remediated before the final project is considered clean.

## Tools Required

- Google ADK 2.x and Gemini `gemini-3.1-flash-lite`.
- Pydantic for strict tool inputs.
- Pytest for deterministic outcome tests.
- Semgrep and pre-commit for local repository gates.
- A local STRIDE threat-model skill.

## Constraints & Safety Rules

- Discount policy is deterministic code, not LLM judgment.
- Codes are single-use and require a registered non-guest user.
- No real or simulated credential-shaped value remains in runnable project code.
- Destructive shell commands fail closed.
- Local-only prototype; no cloud deployment or GitHub integration.
- Do not bypass hooks with `--no-verify`.

## Success Criteria

- Agents CLI recognizes the scaffolded project.
- Unit tests cover successful redemption, reuse, invalid codes, guest users, and malformed inputs.
- The command hook allows safe commands and blocks destructive commands.
- Semgrep catches a controlled vulnerable fixture and passes the remediated application.
- A STRIDE threat model and security-aware implementation plan are present.
- The agent can complete a real local Gemini tool-use smoke test when credentials are available.
