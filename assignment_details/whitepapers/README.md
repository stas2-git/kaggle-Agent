# Whitepapers

The 5-day course shipped a whitepaper each day. This page is the extracted, high-level summary. Read [`notes/`](notes/) for the dense study notes behind each day; open [`details/`](details/) only to verify original wording or check a specific claim.

Read the capstone baseline first: [`../capstone_project/README.md`](../capstone_project/README.md)

## Main Ideas Covered

Each day now has a full study-notes page in [`notes/`](notes/) — mechanisms, terminology, and capstone-applicability bullets pulled from the full whitepaper text, not just named concepts. Read the notes page, not the raw extraction, unless you need to verify exact wording.

| Day | Whitepaper | One-line summary | Notes |
|---|---|---|---|
| 1 | Vibe coding intro | Intent-driven "vibe coding" vs. disciplined "agentic engineering," and the "factory model" where developers design the harnesses that guide autonomous execution. | [`notes/day1_notes.md`](notes/day1_notes.md) |
| 2 | Agent tools & interoperability | Open protocols for a plug-and-play agent ecosystem: MCP for data access, A2A for agent collaboration, A2UI for generative UI, AP2/UCP for machine-to-machine commerce. | [`notes/day2_notes.md`](notes/day2_notes.md) |
| 3 | Agent skills | Avoiding "context rot" with portable Agent Skills — directories built around a `SKILL.md` file — using progressive disclosure so a single agent flexes into many specialist roles. | [`notes/day3_notes.md`](notes/day3_notes.md) |
| 4 | Security & evaluation | Continuous "Effective Trust" via a 7-pillar security architecture: ephemeral sandboxing, slopsquatting defenses, a Red/Blue/Green security triad, and trajectory evaluation. | [`notes/day4_notes.md`](notes/day4_notes.md) |
| 5 | Spec-driven production dev | Spec-Driven Development (SDD): code as disposable, Gherkin specs as source of truth, zero-trust pipelines, and hybrid Policy Servers. | [`notes/day5_notes.md`](notes/day5_notes.md) |
