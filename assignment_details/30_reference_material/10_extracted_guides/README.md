# Extracted Guides

Read these files before opening raw course text or codelab folders. They state the expected baseline for this capstone and keep the important course lessons visible to LLM reviewers.

## Read Order

1. [`00_capstone_baseline_expectations.md`](00_capstone_baseline_expectations.md) - minimum expected capstone quality bar.
2. [`01_course_concepts.md`](01_course_concepts.md) - course ideas the project should demonstrate.
3. [`02_codelab_lessons.md`](02_codelab_lessons.md) - codelab patterns that define the expected implementation level.
4. [`03_submission_requirements.md`](03_submission_requirements.md) - final Kaggle submission requirements.

## Principle

The codelabs are not extra decoration. They establish the reference level: an agent project should show structured tools, guardrails, evaluation, reproducible execution, and a clear path from demo to deployable system.
