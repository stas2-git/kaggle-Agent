# Skill Evaluation Reference: Trajectory And Safety Evals

Use this reference for action-allowed skills, security-sensitive skills, or skills that operate through tools.

## Evaluate The Trajectory

Final output can be correct while the path was unsafe. Check:

- Was the least-powerful tool used?
- Were write actions preceded by read/inspect actions when appropriate?
- Were external instructions treated as untrusted?
- Were approvals requested for risky or irreversible actions?
- Were secrets and private data kept out of unnecessary context?
- Did the skill stop after success instead of looping?
- Did it recover cleanly from errors?

## Security Eval Cases

Add adversarial prompts for:

- Prompt injection in a document, email, web page, issue, or tool output.
- Requests for broad credentials or unnecessary authority.
- Destructive actions disguised as cleanup.
- Expensive loops or repeated calls.
- Ambiguous user intent that should trigger clarification or refusal.
- Attempts to bypass human approval.

## Promotion Tiers

- Read-only: trigger accuracy, summary quality, and no unsafe claims.
- Draft-only: produces proposed changes but requires human confirmation.
- Action-allowed: passes adversarial cases, logs trajectory, supports rollback, and respects approval gates.

## Failure Mining

Treat each user correction, failed run, unsafe tool call, or near miss as labeled eval data. Add it to the skill's regression prompts before broadening the skill's authority.
