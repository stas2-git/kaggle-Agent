# Context Harness Debugger Reference: Skill And Context Failures

Use this reference when the failure involves skill routing, over-loaded context, missing procedural memory, or conflicting instructions.

## Skill Failure Modes

- Trigger failure: the right skill did not load, or the wrong skill loaded.
- Execution failure: the skill loaded but did not tell the agent what to do.
- Context failure: the skill loaded too much background and crowded out task details.
- Composition failure: multiple skills overlapped or contradicted each other.
- Regression failure: a new or edited skill stole traffic from another skill.

## Progressive Disclosure Check

For the failing skill or instruction set:

- Is the description specific enough to route?
- Does `SKILL.md` contain procedure rather than background?
- Are long details moved into `references/`?
- Are deterministic repeated steps moved into `scripts/`?
- Are references named by when to read them?
- Can the agent succeed without loading every reference?

## Debugging Actions

1. Write the user prompt that should have triggered the behavior.
2. Identify the skill or rule that should have handled it.
3. Compare the prompt to the skill description.
4. Decide whether to edit the trigger, the workflow, the reference, or the eval.
5. Add one positive trigger prompt and one negative prompt to prevent regression.

## Red Flags

- The fix is "add more ALWAYS instructions."
- A reference file is a copied source document rather than an operating guide.
- The skill has multiple unrelated jobs.
- The agent must read a thousand lines to perform a five-step workflow.
- The skill says what matters but not what action to take.
