# Agent Self-Audit Reference: Harness And Verification

Use this reference to check whether the agent is falling into the "fast plausible draft" trap. The point is not to repeat the whitepaper; it is to force a better final check before handing work back.

## Audit Prompts

- What did the user actually ask for, including later corrections?
- What would make the answer look plausible while still being wrong?
- Did I verify the artifact, or only reason about it?
- Did I check the path taken: tools, files, assumptions, and side effects?
- Did I overuse broad context instead of loading the smallest useful source?
- Did I blame model quality for something caused by weak instructions, missing tools, missing tests, or bad context?

## Common Agent Weaknesses

- Stops at the first working-looking answer.
- Treats generated code or prose as evidence.
- Misses edge cases because the request was vague.
- Lets long context override the newest user instruction.
- Hallucinates paths, dependencies, APIs, or project state.
- Produces a nice summary without concrete verification.

## Stronger Behavior

- Reconstruct intent in one sentence.
- Name the evidence used to verify the result.
- Inspect the action trajectory when tools were used.
- Convert uncertainty into a test, file read, command, or explicit residual risk.
- Prefer a small fix with proof over a broad confident rewrite.

## Final-Handoff Check

Before final response, complete this compact audit:

```text
Intent: What outcome did the user want?
Verified: What concrete check did I run or inspect?
Trajectory: Were tool calls/files/actions appropriate and scoped?
Weak Spots: What plausible failure did I check?
Residual Risk: What remains unverified, if anything?
```
