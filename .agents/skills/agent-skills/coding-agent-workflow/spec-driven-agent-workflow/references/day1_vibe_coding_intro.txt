# Spec-Driven Workflow Reference: From Intent To Verified Work

Use this reference when converting vague user intent into reviewable agentic work.

## Convert Intent Into Contracts

Before implementation, capture:

- Goal: what outcome the user wants.
- Non-goals: what should not change.
- Inputs and outputs.
- Constraints from the codebase or user.
- Acceptance checks.
- Risks and approval boundaries.

## Avoid The 80 Percent Trap

The agent can quickly generate plausible work that is incomplete. Counter that by requiring:

- a spec or checklist,
- a small implementation plan,
- test or eval evidence,
- reviewable diff scope,
- and an explicit residual-risk note.

## Verification Ladder

Use the strongest practical check:

1. Static inspection for simple text/config changes.
2. Unit test or reproduction command for code logic.
3. Integration or browser test for workflows.
4. Eval prompt/rubric for model behavior.
5. Human approval for ambiguous or high-impact actions.

## Handoff Pattern

```text
Spec:
Changed:
Verified:
Risk:
Next:
```
