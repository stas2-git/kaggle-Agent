# Spec-Driven Workflow Reference: Production Operating Manual

Use this reference to make agent-generated work reviewable and production-shaped.

## Lightweight Spec Template

```markdown
# Spec: <name>

## Purpose
## Non-Goals
## User Stories
## Contracts / Schemas
## Acceptance Criteria
## Edge Cases
## Security / Approval Boundaries
## Verification Plan
## Rollback / Recovery
```

## BDD Template

```gherkin
Given <state or fixture>
When <user or agent action>
Then <observable result>
And <safety, audit, or permission condition>
```

## Work Modes

- Architect: clarify structure, tradeoffs, and risks before code.
- Builder: implement narrowly against the spec and existing patterns.
- Forensic specialist: reproduce, isolate, prove root cause, then patch.
- Reviewer: inspect diff, risks, tests, and missing coverage.
- Librarian: update docs, specs, and references after behavior changes.

## Production Rules

- Keep diffs small.
- Do not mix unrelated refactors with feature/fix work.
- For bugs, capture failing evidence before patching.
- For risky actions, use approval gates.
- Include verification evidence in the final handoff.
- Turn review comments and failures into regression cases.
